feat: Add persistent chat session support for Gumloop proxy

PROBLEM:
Every request to Gumloop created a new chat session (interaction_id),
causing AI to lose context between messages. Users experienced "newchat terus"
- each prompt started a fresh conversation with no memory of previous turns.

ROOT CAUSE:
1. proxy_gumloop.py always generated new interaction_id per request
2. interaction_id only sent to Gumloop when images were present
3. No database persistence for interaction_id across requests

SOLUTION:
1. Database:
   - Add `gumloop_interaction_id` column to `chat_sessions` table
   - Add migration for existing databases
   - New helper: `get_or_create_gumloop_interaction_id(session_id)`

2. Proxy (unified/proxy_gumloop.py):
   - Accept `chat_session_id` parameter in request body
   - Retrieve/create persistent interaction_id for that session
   - Always send interaction_id to Gumloop (removed conditional)

USAGE:
```python
# Request with persistent session
response = client.chat.completions.create(
    model="gl-claude-sonnet-4-5",
    messages=[{"role": "user", "content": "My name is Alice"}],
    chat_session_id=1  # ← Enables persistence
)

# Next request with same session_id remembers context
response = client.chat.completions.create(
    model="gl-claude-sonnet-4-5",
    messages=[{"role": "user", "content": "What's my name?"}],
    chat_session_id=1  # ← Same session
)
# AI responds: "Your name is Alice" ✅
```

BACKWARD COMPATIBILITY:
✅ Fully backward compatible
- Requests without `chat_session_id` work as before (one-off chats)
- Requests with `chat_session_id` get persistent sessions
- No breaking changes to existing clients
